# Support

## Getting help

Start with the docs, in this order:

1. **`README.md`** — what this is, the two installation profiles (MINIMAL/MULTI), and the demo path.
2. **`INIT.md`** — the guided install itself; paste it into your agent CLI.
3. **`docs/`** — task-specific detail: `docs/what-gets-written.md` (every file this project writes), `docs/uninstall.md`, `docs/upgrade.md`, `docs/sync-contract.md`, `docs/council.md`, `docs/team.md` (if you're evaluating this for more than one person), `docs/lazy-skills.md`.
4. **`SECURITY.md`** — for a vulnerability report specifically; do not open a public issue for anything that could expose a real credential or an active exploit path.

If none of that answers it, [open an issue on GitHub](https://github.com/matteopasseri407/NeXgen-Engine/issues). That is the real, current support channel: there is no separate community chat or forum today, so issues are where bug reports and usage questions actually get looked at.

# Supporto

## Come ottenere aiuto

Parti dalla documentazione, in quest'ordine:

1. **`README.md`** — cos'è il progetto, i due profili di installazione (MINIMAL/MULTI) e il percorso demo.
2. **`INIT.md`** — l'installazione guidata vera e propria; incollalo nella tua CLI agentica.
3. **`docs/`** — dettagli specifici: `docs/what-gets-written.md` (ogni file che questo progetto scrive), `docs/uninstall.md`, `docs/upgrade.md`, `docs/sync-contract.md`, `docs/council.md`, `docs/team.md` (se lo stai valutando per più di una persona), `docs/lazy-skills.md`.
4. **`SECURITY.md`** — per segnalare una vulnerabilità nello specifico; non aprire un issue pubblico per qualunque cosa possa esporre una credenziale reale o un exploit attivo.

Se nessuno di questi risponde alla tua domanda, [apri un issue su GitHub](https://github.com/matteopasseri407/NeXgen-Engine/issues). È il canale di supporto reale e attuale: oggi non esiste una chat o un forum della community separati, quindi gli issue sono il punto dove bug e domande d'uso vengono davvero seguiti.
